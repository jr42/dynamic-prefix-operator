== Getting Started ==

1. Create a DynamicPrefix resource to start receiving IPv6 prefixes:

   # Mode 1: Address Range (Recommended for home/SOHO)
   # Reserve a range within your existing /64
   kubectl apply -f - <<EOF
   apiVersion: dynamic-prefix.io/v1alpha1
   kind: DynamicPrefix
   metadata:
     name: my-prefix
   spec:
     acquisition:
       routerAdvertisement:
         interface: eth0
         enabled: true
     addressRanges:
       - name: loadbalancers
         start: "::f000:0:0:0"
         end: "::ffff:ffff:ffff:ffff"
   EOF

2. Check the status of your DynamicPrefix:

   kubectl get dynamicprefixes
   kubectl describe dynamicprefix my-prefix

3. Add annotations to Cilium resources you want to sync:

   For CiliumLoadBalancerIPPool (using address range):

   kubectl annotate ciliumloadbalancerippool my-pool \
     dynamic-prefix.io/name=my-prefix \
     dynamic-prefix.io/address-range=loadbalancers

   For CiliumCIDRGroup:

   kubectl annotate ciliumcidrgroup my-group \
     dynamic-prefix.io/name=my-prefix \
     dynamic-prefix.io/address-range=loadbalancers

== Supported Annotations ==

- dynamic-prefix.io/name: References the DynamicPrefix CR
- dynamic-prefix.io/address-range: References an address range (Mode 1)
- dynamic-prefix.io/subnet: References a subnet (Mode 2)
